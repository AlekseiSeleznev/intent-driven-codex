## Why

A project validation report found that the functional overlay and specs pass, but public documentation, installation instructions, version metadata, and a few context docs have drifted from the repository's current OpenSpec/ADR/Claude-review state. Several findings, especially Greenfield and manual install instructions, can lead users to a broken installation.

## What Changes

- Fix README/README.ru state drift: active/archived changes, in-force ADR list, repository structure, command table, and `c4-diagrams` naming.
- Fix installation guidance so Greenfield uses `--force-overlay`, manual copy instructions match the installer/check-overlay requirements, and duplicate unsafe `cp` steps are removed or rewritten.
- Align repository `openspec/config.yaml` with the richer configuration generated by `scripts/install-overlay`.
- Update release metadata by bumping `VERSION`, refreshing README badges/state text, and adding a `CHANGELOG.md` entry for recent 2026-05-30 features.
- Polish context docs: `scripts/check-overlay` spelling, ADR 0007 supersession wording, and full `design` dependencies in lifecycle docs.
- Keep `openspec/changes/fix-validation-report-findings/validation-report.md` as the non-secret source report for this change.

## Capabilities

- Modify `template-installation` for install and documentation accuracy requirements.
- Modify `intent-driven-schema` for repository `openspec/config.yaml` alignment with generated intent-driven configuration.
- Modify `project-context-architecture` for architecture/ADR documentation coherence.
- Modify `codex-opsx-workflow` for `/opsx:*` command documentation coverage.
- Modify `overlay-update-safety` for documentation drift prevention and overlay check spelling/coverage expectations.

## Impact

- Affected docs: `README.md`, `README.ru.md`, `INSTALL_CODEX.md`, `docs/update-safety.md`, `docs/lifecycle.md`, `ARCHITECTURE.md`, `adr/README.md`, and new `CHANGELOG.md`.
- Affected config/version files: `openspec/config.yaml`, `VERSION`, and README version badges/state sections.
- Affected verification: OpenSpec validation, schema validation, full `scripts/check-overlay`, install-path smoke checks where practical, documentation grep consistency checks, tracked-secret checks, and git status.
- No external systems or secrets are required; `.secrets.local.env` must not be read.
- Architecture constraints: preserve OpenSpec CLI boundary, project-local overlay ownership, append-only durable ADR history, no-secret policy, and automatic local checkpoint discipline.
